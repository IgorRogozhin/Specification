Desktop-приложение "Спецификация" ("Specification")
Приложение, которое помогает составлять документы спецификации 
согласно ЕСКД.

Функционал :
1) Выбор подготовленного в NX (Unigraphics) txt-файла 
2) Замена стандартных элементов на перечень из СТП
3) Сохранение xls-файла в папку с исходным файлом
4) Множественный выбор файлов

Как это работает и запускается:
1) Установить в системе JRE 8
2) В папке "Программа для спецификаций" запустить исполняемый
jar-файл "Запускай", выбрать заранее подготовленный txt-файл
(e.g. БЛ2.565.001СБ.txt, БЛ3.619.019СБ.txt, БЛ6.177.517.txt) 
3) Выбрать из предлагаемых вариантов стандартных элементов
4) Открыть получившийся файл с тем же названием, но в формате xls.

Архитектура проекта:
Проект работает в локальной сети, где на общем диске есть
папка "GOST_doNotRemove", там лежат 2 файла после сериализации:
-  GOST.out представляет собой таблицы "Стандартные изделия.xlsx",
чтобы каждый раз не читать их заново
-  ReplaceForGOST.out - представляет собой алгоритмы по поиску и 
подбору подходящих вариантов замены для стандартных элементов.
В случае расширения перечня стандартных элементов, просто заменяются
эти два файла, без необходимости скачивания самого jar-файла.
В случае если сетевой ресурс недоступен (например, как Вам сейчас),
то эти два файла считываются из самого архива, но стандартные элементы
там будут лишь те, которые существовали на момент скачивания.
